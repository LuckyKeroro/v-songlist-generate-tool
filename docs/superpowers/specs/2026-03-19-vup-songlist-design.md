# Vup/Vtuber 歌单网站设计文档

## 项目概述

为虚拟主播（Vup/Vtuber）设计的歌单网站，包含本地管理工具和静态歌单网站两部分。用户通过管理工具维护歌曲元数据，生成静态网站后手动部署到 GitHub Pages。

## 架构设计

### 目录结构

```
krr-songlist/
├── admin/                    # 本地管理工具
│   ├── index.html           # 管理界面
│   ├── server.js            # Express 服务器
│   └── package.json
├── site/                     # 静态网站源码 (React + Vite)
│   ├── src/
│   │   ├── components/      # React 组件
│   │   ├── App.jsx
│   │   └── main.jsx
│   ├── package.json
│   └── vite.config.js
├── Resources/               # 歌曲元数据（JSON）
│   └── {歌手}/
│       └── {专辑}/
│           └── {歌曲}.json
├── public/                  # 静态资源
│   ├── background.jpg       # 背景图
│   ├── avatar.jpg           # 头像
│   └── config.json          # 网站配置
└── dist/                    # 生成的静态网站输出
```

## 本地管理工具

### 运行方式

Node.js 本地服务器，通过 `npm run admin` 启动，访问 `http://localhost:3001`。

### 功能模块

#### 1. 歌曲管理区

- 上传 Excel 按钮，支持拖拽上传
- 解析后显示歌曲列表（歌名、歌手、状态）
- 状态：待搜索 / 已确认 / 手动输入
- 「搜索元数据」按钮：调用网易云 API，显示搜索结果供选择
- 「手动编辑」按钮：弹窗填写完整信息（搜索不到时使用）

#### 2. 视觉配置区

- 背景图上传（支持拖拽）
- 头像上传（支持拖拽）
- 实时预览缩略图

#### 3. 链接配置区

- 链接1：显示文字 + URL
- 链接2：显示文字 + URL

#### 4. 公告配置区

- 文本框输入自定义公告内容

#### 5. 生成区

- 「生成静态网站」按钮：生成到 `dist/`
- 「本地预览」按钮：启动预览服务器

### 技术栈

- 后端：Express.js
- 前端：原生 HTML + CSS + JavaScript
- Excel 解析：SheetJS (xlsx)
- 网易云 API：NeteaseCloudMusicApi

## 静态歌单网站

### 页面布局

```
┌─────────────────────────────────────────────────────┐
│                    自定义背景图                        │
│  ┌─────────────────────────────────────────────┐    │
│  │           毛玻璃容器 (宽度 61.8%)             │    │
│  │  ┌───────────────────────────────────────┐  │    │
│  │  │ ┌──────┐ ┌─────────────────────────┐  │  │    │
│  │  │ │      │ │[链接1] [链接2] (白底)    │  │  │    │
│  │  │ │ 头像 │ │─────────────────────────│  │  │    │
│  │  │ │(白底)│ │  公告区域 (白底)         │  │  │    │
│  │  │ │      │ │                         │  │  │    │
│  │  │ └──────┘ └─────────────────────────┘  │  │    │
│  │  │     主播名字 (白底)                   │  │    │
│  │  ├───────────────────────────────────────┤  │    │
│  │  │  [搜索框]     [语言筛选 ▼]  (白底)    │  │    │
│  │  ├───────────────────────────────────────┤  │    │
│  │  │  歌曲列表 (按歌手首字母分组)            │  │    │
│  │  │  ── A ──                               │  │    │
│  │  │  歌曲名 - 歌手   专辑    语言          │  │    │
│  │  └───────────────────────────────────────┘  │    │
│  └─────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

### 视觉风格

- 毛玻璃容器：`backdrop-filter: blur(20px)` + `background: rgba(255,255,255,0.15)`
- 内部元素：白色背景 + 圆角，形成层次分明的卡片式设计
- 响应式设计，移动端容器宽度自适应

### 交互功能

- **搜索**：实时过滤，匹配歌名、歌手、歌词
- **语言筛选**：下拉选择（全部 / 中文 / 日语 / 英语 / 韩语）
- **歌曲点击**：复制 `点歌 {歌名} - {歌手}` 到剪贴板，显示 toast 提示
- **分组显示**：按歌手首字母（A-Z + #）分组，支持中英文混合排序

### 技术栈

- 框架：React 18 + Vite
- 样式：CSS Modules
- 拼音排序：pinyin-pro

## 数据格式

### 歌曲元数据 (Resources/{歌手}/{专辑}/{歌曲}.json)

```json
{
  "title": "光るなら",
  "artist": "Goose house",
  "album": "光るなら",
  "releaseDate": "2014-11-19",
  "lyrics": "雨上がりの空を...",
  "language": "日语",
  "cover": "cover.jpg"
}
```

### 网站配置 (public/config.json)

```json
{
  "name": "星离Starli",
  "avatar": "avatar.jpg",
  "background": "background.jpg",
  "links": [
    { "text": "B站空间", "url": "https://space.bilibili.com/xxx" },
    { "text": "B站直播", "url": "https://live.bilibili.com/xxx" }
  ],
  "announcement": "欢迎来到我的歌单！点歌请点击歌曲名称复制~"
}
```

## 生成流程

1. 读取 `Resources/` 下所有 JSON 文件
2. 读取 `public/config.json` 配置
3. 复制资源文件到 `dist/`
4. 打包 React 应用，注入数据
5. 输出到 `dist/`

## 开发命令

```bash
# 安装依赖
npm install

# 启动管理工具 (http://localhost:3001)
npm run admin

# 开发静态网站
npm run dev

# 生成静态网站到 dist/
npm run generate
```

## 部署说明

生成后的 `dist/` 文件夹内容可手动上传到 GitHub Pages 或其他静态托管服务。

## Excel 输入格式

Excel 表格支持混合格式：
- 部分行：歌名 + 歌手
- 部分行：只有歌名

系统会根据可用信息智能搜索网易云音乐 API 获取元数据。
